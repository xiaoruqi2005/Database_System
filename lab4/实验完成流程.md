# Lab 4 实验完成流程

## 1. 实验目标
根据 lab4.md，实现 DATETIME 时间类型（8字节），格式为 'YYYY-MM-DD HH:MM:SS'，范围 1000-01-01 00:00:00 ~ 9999-12-31 23:59:59，支持增删改查和输入合法性校验。

## 2. 需求拆解
- 新增 TYPE_DATETIME 类型和 SV_TYPE_DATETIME 解析类型
- 修改 lexer/parser 支持 DATETIME 关键字
- 实现 DATETIME 字符串解析和合法性校验（年月日时分秒范围、闰年、格式长度）
- 将 DATETIME 编码为 long long（YYYYMMDDHHMMSS）存储，数值顺序等于时间顺序
- 修改所有 switch(type) 分支支持 DATETIME 的比较和输出
- 支持 INSERT/UPDATE/DELETE/WHERE 路径的 STRING->DATETIME 隐式转换

## 3. 实现思路
### 3.1 类型系统
- ColType 新增 TYPE_DATETIME，SvType 新增 SV_TYPE_DATETIME
- Value 联合体新增 datetime_val (long long)，set_datetime() 方法
- init_raw() 新增 TYPE_DATETIME 分支，以 long long 序列化

### 3.2 存储编码
- DATETIME 编码为 long long: YYYY*10^10 + MM*10^8 + DD*10^6 + HH*10^4 + MM*10^2 + SS
- 数值大小关系与时间先后一致，可直接用 long long 比较

### 3.3 合法性校验
- 格式必须为 19 字符 'YYYY-MM-DD HH:MM:SS'
- 年: 1000~9999，月: 1~12，日: 1~当月最大天数（含闰年2月）
- 时: 0~23，分: 0~59，秒: 0~59
- 非法输入抛出 IncompatibleTypeError，服务端输出 failure

### 3.4 类型兼容性
- STRING -> DATETIME 隐式转换（INSERT/UPDATE/WHERE 路径）
- DATETIME 列接受字符串值，解析并校验后存储为 long long

### 5.2 测试点1 - DATETIME 增删改查
```
CREATE TABLE t(id int , time datetime);
INSERT INTO t values(1, '2023-05-18 09:12:19');
INSERT INTO t values(2, '2023-05-30 12:34:32');
SELECT * FROM t;
DELETE FROM t WHERE time = '2023-05-30 12:34:32';
UPDATE t SET id = 2023 WHERE time = '2023-05-18 09:12:19';
SELECT * FROM t;
```
结果: **PASS** - 输出与预期完全一致

### 5.3 测试点2 - 合法性校验
```
CREATE TABLE t(time datetime, temperature float);
INSERT INTO t values('1999-07-07 12:30:00' , 36.0);  -- ok
INSERT INTO t values('1999-13-07 12:30:00' , 36.0);  -- failure (month=13)
INSERT INTO t values('1999-1-07 12:30:00' , 36.0);   -- failure (format)
INSERT INTO t values('1999-00-07 12:30:00' , 36.0);  -- failure (month=0)
INSERT INTO t values('1999-07-00 12:30:00' , 36.0);  -- failure (day=0)
INSERT INTO t values('0001-07-10 12:30:00' , 36.0);  -- failure (year<1000)
INSERT INTO t values('1999-02-30 12:30:00' , 36.0);  -- failure (Feb 30)
INSERT INTO t values('1999-02-28 12:30:61' , 36.0);  -- failure (second=61)
```
结果: **PASS** - 7 个非法输入均输出 failure，合法数据正确存储
