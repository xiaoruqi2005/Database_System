# Lab2 实验完成流程

## 1. 实验目标

根据 `lab2.md` 要求，在实验一（存储管理）的基础上增加查询执行模块，使系统支持元数据管理、DDL语句（create table / drop table）、DML语句（insert / delete / update）和DQL语句（select）。

本实验包含五个测试点：
- 测试点1：建表与删表（DDL）
- 测试点2：单表插入与条件查询
- 测试点3：单表更新与条件查询
- 测试点4：单表删除与条件查询
- 测试点5：连接查询

## 2. 需求拆解

- 子任务1：实现 UpdateExecutor 的更新逻辑（原代码为空）
- 子任务2：修复 NestedLoopJoinExecutor 的迭代器逻辑（火山模型接口错误）
- 子任务3：实现 DeleteExecutor 的索引维护
- 子任务4：实现 TransactionManager 的最小化事务支持（begin/commit/abort）
- 子任务5：修复 SetTransaction 在事务提交后的崩溃问题
- 子任务6：修复 drop_table 不正确关闭文件句柄的问题
- 子任务7：修复 WHERE 条件中 INT/FLOAT 类型比较和隐式转换
- 子任务8：修复 STRING 比较使用 strncmp 避免越界读取
- 子任务9：修复多表 JOIN 的连接顺序

## 3. 实现思路

### 3.1 UpdateExecutor（更新算子）

- **原有流程**：`Next()` 直接返回 `nullptr`，没有任何更新逻辑。
- **实现方案**：
  1. 遍历所有待更新的 Rid
  2. 读取旧记录
  3. 删除旧记录的索引条目（如有索引）
  4. 应用 SET 子句修改字段值
  5. 调用 `update_record` 写回记录
  6. 插入新记录的索引条目
- **关键数据结构**：`SetClause`（列名+值）、`TabMeta`（表元数据，含索引信息）
- **边界条件**：无索引时跳过索引维护；多列更新时逐列处理

### 3.2 NestedLoopJoinExecutor（嵌套循环连接算子）

- **原有流程**：混淆了火山模型的 `Next()` 和 `nextTuple()` 语义，导致游标状态混乱。
- **实现方案**：
  1. 重写为正确的火山模型：`beginTuple()` 初始化双循环，`nextTuple()` 推进右表游标
  2. `find_next_valid()` 负责寻找满足 join 条件的记录对
  3. `Next()` 只返回当前游标位置的拼接记录，不移动游标
  4. `is_end()` 使用 `isend` 标志判断是否遍历完成
- **边界条件**：空表、无条件笛卡尔积、单条件等值连接

### 3.3 DeleteExecutor（删除算子）

- **原有流程**：只调用 `delete_record`，不维护索引。
- **实现方案**：删除记录前，先读取记录内容，从所有索引中删除对应的索引条目，然后再删除记录。

### 3.4 TransactionManager（事务管理器）

- **原有流程**：`begin()` 返回 `nullptr`，`commit()` 和 `abort()` 为空实现，导致服务端崩溃。
- **实现方案**：
  1. `begin()`：创建新 Transaction 对象，分配递增的 txn_id 和时间戳，加入全局事务表
  2. `commit()`：设置事务状态为 COMMITTED，从全局事务表中移除
  3. `abort()`：设置事务状态为 ABORTED，从全局事务表中移除

### 3.5 SetTransaction（事务管理）

- **原有流程**：单条 SQL 隐式事务提交后，`txn_id` 仍保留旧值，下次调用 `get_transaction()` 触发 assert 崩溃。
- **实现方案**：在 commit 和 abort 之后将 `txn_id` 重置为 `INVALID_TXN_ID`，使下次循环创建新事务。

### 3.6 drop_table（删除表）

- **原有流程**：`fhs_.erase()` 只销毁了 unique_ptr，但没有调用 `close_file()` 关闭文件描述符，导致 `destroy_file()` 报错。
- **实现方案**：先调用 `rm_manager_->close_file()` 正确刷盘并关闭 fd，再从 map 移除，最后删除文件。

### 3.7 WHERE 条件类型检查与隐式转换

- **原有流程**：WHERE 中 FLOAT 列与 INT 值比较时直接抛出类型不匹配异常。
- **实现方案**：
  1. 在 `check_clause` 中允许 INT/FLOAT 之间的比较
  2. 在 `init_raw` 前进行隐式类型转换（INT→FLOAT 或 FLOAT→INT）
  3. 修复了 `rhs_type` 变量在类型转换前未初始化的 bug

### 3.8 STRING 比较修复

- **原有流程**：`compare_value` 使用 `strcmp` 比较 STRING 类型，但 `char(n)` 字段在字符串长度等于 n 时没有 null 终止符，导致越界读取。
- **实现方案**：将 `strcmp` 改为 `strncmp(lhs, rhs, len)`，传入列长度参数。

### 3.9 JOIN 连接顺序

- **原有流程**：planner 中无条件多表 JOIN 时，剩余表被放在左子节点，导致列顺序和行顺序与预期不符。
- **实现方案**：将已连接的表保持在左子节点，新表放在右子节点。
