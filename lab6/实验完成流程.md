# Lab6 实验完成流程

## 1. 实验目标
- 根据 lab6.md 要求，在 RMDB 中实现 SQL 聚合函数 SUM、MAX、MIN、COUNT
- 支持 `COUNT(*)`、`COUNT(col)`、`SUM(col)`、`MAX(col)`、`MIN(col)`
- 支持 `AS` 别名语法
- 支持与 `WHERE` 条件组合使用
- 浮点数保留 6 位小数，整数不显示小数

## 2. 需求拆解
- 子任务 1：词法分析器添加 SUM/MAX/MIN/COUNT/AS 关键字
- 子任务 2：语法分析器添加聚合函数表达式和 SELECT 聚合查询语法规则
- 子任务 3：AST 中新增 AggExpr 节点和 AggType 枚举
- 子任务 4：语义分析器解析聚合表达式，确定结果类型和别名
- 子任务 5：查询计划层新增 AggregatePlan
- 子任务 6：执行引擎新增 AggregateExecutor（阻塞式聚合算子）
- 子任务 7：Portal 层适配 AggregatePlan 到 AggregateExecutor 的转换

## 3. 实现思路

### 3.1 聚合查询语法解析
- 原有流程：`SELECT selector FROM ...` 中 `selector` 仅支持 `*` 或列列表
- 实现方案：新增独立的 `agg_list` 语法规则，支持 `SUM(col) as alias` 形式
- 关键数据结构：`ast::AggExpr` 包含 `agg_type`、`is_count_star`、`col`、`alias`
- SELECT 规则新增分支：`SELECT agg_list FROM tableList optWhereClause opt_order_clause`

### 3.2 语义分析
- 原有流程：SelectStmt 的 `cols`（TabCol 列表）用于投影
- 实现方案：新增 `AggSel` 结构体描述聚合选择，存储在 `Query.agg_sels` 中
- 类型推导规则：
  - COUNT(*), COUNT(col) → TYPE_INT
  - SUM(col) → 参数列类型（仅支持 int/float）
  - MAX(col), MIN(col) → 参数列类型（支持 int/float/char）

### 3.3 阻塞式聚合算子
- 原有流程：所有执行器都是火山模型（迭代式 pull）
- 实现方案：AggregateExecutor 采用阻塞式设计，在 `beginTuple()` 中消费所有子算子的行并计算结果，然后只输出一行
- 聚合逻辑：
  - COUNT：行计数
  - SUM：int 累加到 long long，float 累加到 double
  - MAX/MIN：逐行比较取极值，支持 int/float/bigint/string

### 3.4 输出格式
- `select_from()` 通过 `cols()` 获取 ColMeta 格式化输出
- AggregateExecutor 返回合成的 ColMeta（name=别名, type=结果类型）
- float 类型通过 `std::to_string()` 输出，默认 6 位小数，符合题目要求

## 4. 修改文件明细

### `rmdb/src/parser/ast.h`
- 新增 `AggType` 枚举（AGG_SUM, AGG_MAX, AGG_MIN, AGG_COUNT）
- 新增 `AggExpr` AST 节点结构体
- `SelectStmt` 新增 `aggs` 字段
- `SemValue` 新增 `sv_agg_type`、`sv_agg`、`sv_aggs` 字段

### `rmdb/src/parser/lex.l`
- 新增关键字：SUM、MAX、MIN、COUNT、AS

### `rmdb/src/parser/yacc.y`
- 新增 token 声明：SUM MAX MIN COUNT AS
- 新增类型声明：agg_func、agg_expr、agg_list、opt_as_alias
- 新增 SELECT 聚合查询语法分支
- 新增聚合函数语法规则（agg_func、opt_as_alias、agg_expr、agg_list）

### `rmdb/src/common/common.h`
- 新增 `AggSel` 结构体（agg_type、is_count_star、col、alias、agg_col_type）
- 新增 `#include "parser/ast.h"` 以使用 `ast::AggType`

### `rmdb/src/analyze/analyze.h` / `analyze.cpp`
- `Query` 类新增 `agg_sels` 字段
- `do_analyze` 方法新增聚合表达式处理逻辑：解析参数列、推导结果类型、设置别名

### `rmdb/src/optimizer/plan.h`
- `PlanTag` 枚举新增 `T_Aggregate`
- 新增 `AggregatePlan` 类（subplan_、aggs_）

### `rmdb/src/optimizer/planner.cpp`
- `generate_select_plan` 方法修改：检测 `agg_sels` 非空时创建 `AggregatePlan`，否则创建 `ProjectionPlan`

### `rmdb/src/execution/executor_aggregate.h`（新建）
- `AggregateExecutor` 类：阻塞式聚合执行器
- `beginTuple()` 消费所有子行，计算 SUM/MAX/MIN/COUNT
- `Next()` 返回单行聚合结果
- `cols()` 返回合成 ColMeta（别名作为列名）

### `rmdb/src/portal.h`
- 新增 `#include "execution/executor_aggregate.h"`
- `start` 方法 T_select 分支：检测 AggregatePlan，构建别名作为 sel_cols
- `convert_plan_executor` 方法：新增 AggregatePlan → AggregateExecutor 转换
