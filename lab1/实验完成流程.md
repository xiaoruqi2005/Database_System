# 实验一完成流程

## 一、实验目标

本实验基于 RMDB 框架完成存储管理模块，主要补全磁盘管理器、缓冲池管理器、LRU 替换器和记录管理器，使系统能够正确完成页读写、页缓存替换、记录插入/删除/更新/扫描等基础存储功能。

实验对应的核心模块包括：

- `src/storage/disk_manager.cpp`
- `src/storage/buffer_pool_manager.cpp`
- `src/replacer/lru_replacer.cpp`
- `src/record/rm_file_handle.cpp`
- `src/record/rm_file_handle.h`
- `src/record/rm_scan.cpp`
- `src/common/config.h`

## 二、总体实现思路

实验一的整体思路是自底向上补齐存储层：

1. 先实现 `DiskManager`，保证文件创建、打开、关闭、删除和页读写可用。
2. 再实现 `LRUReplacer`，为缓冲池提供可淘汰页框选择能力。
3. 然后实现 `BufferPoolManager`，负责内存页框、磁盘页、页表、空闲链表、脏页回写和 pin count 的协同管理。
4. 最后实现 `RmFileHandle` 和 `RmScan`，在页管理之上实现记录级操作。

其中，缓冲池中所有被使用的页都需要通过 `pin_count_` 防止被淘汰；当 `pin_count_` 下降到 0 时，再交给 LRU 替换器管理。记录页通过 bitmap 标识槽位是否被占用，通过文件头中的 `first_free_page_no` 维护仍有空闲槽的页面链表。

## 三、具体修改内容

### 1. `src/storage/disk_manager.cpp`

完成磁盘文件和页读写相关接口。

修改点：

- 实现 `DiskManager::write_page`
  - 使用 `lseek(fd, page_no * PAGE_SIZE, SEEK_SET)` 定位页起始位置。
  - 使用 `write` 写入指定字节数。
  - 若写入字节数不等于 `num_bytes`，抛出 `InternalError`。

- 实现 `DiskManager::read_page`
  - 使用 `lseek` 定位页偏移。
  - 使用 `read` 读取指定字节数。
  - 若读取字节数不等于 `num_bytes`，抛出 `InternalError`。

- 实现 `DiskManager::create_file`
  - 创建前检查文件是否已存在。
  - 使用 `open(..., O_CREAT | O_EXCL | O_RDWR, ...)` 创建文件。
  - 创建后立即关闭文件描述符。

- 实现 `DiskManager::destroy_file`
  - 删除前检查文件是否存在。
  - 若文件仍在打开表 `path2fd_` 中，则抛出 `FileNotClosedError`。
  - 使用 `unlink` 删除文件。

- 实现 `DiskManager::open_file`
  - 检查文件是否存在。
  - 防止重复打开同一文件。
  - 使用 `open(..., O_RDWR)` 打开文件。
  - 维护 `path2fd_` 和 `fd2path_` 映射。

- 实现 `DiskManager::close_file`
  - 检查 fd 是否已打开。
  - 调用 `close` 关闭文件。
  - 从 `path2fd_` 和 `fd2path_` 中删除映射。

### 2. `src/replacer/lru_replacer.cpp`

实现 LRU 缓冲池替换策略。

修改点：

- 实现 `LRUReplacer::victim`
  - 当 LRU 链表为空时返回 `false`。
  - 从链表头部取出最久未使用的 frame。
  - 同步删除哈希表中的 frame 记录。

- 实现 `LRUReplacer::pin`
  - 表示该 frame 正在使用，不能被替换。
  - 如果 frame 当前在 LRU 链表中，则从链表和哈希表中移除。

- 实现 `LRUReplacer::unpin`
  - 表示该 frame 不再被使用，可以被替换。
  - 若 frame 已存在于 LRU 中则不重复插入。
  - 将 frame 插入链表尾部。

实现中使用 `std::scoped_lock` 保证替换器内部结构线程安全。

### 3. `src/storage/buffer_pool_manager.cpp`

实现缓冲池页面调度、换入换出和刷盘。

修改点：

- 实现 `find_victim_page`
  - 优先从 `free_list_` 获取空闲 frame。
  - 若无空闲 frame，则调用 `replacer_->victim` 获取可淘汰 frame。

- 实现 `update_page`
  - 如果旧页有效且为脏页，则写回磁盘。
  - 从页表中删除旧页映射。
  - 重置页内存、页号、脏页标记和 pin count。
  - 若新页有效，则建立新页到 frame 的映射。

- 实现 `fetch_page`
  - 若目标页已在缓冲池中，增加 `pin_count_` 并从 replacer 中 pin。
  - 若不在缓冲池中，寻找 victim frame。
  - 必要时写回旧脏页。
  - 从磁盘读取目标页到 frame。
  - 设置新页 pin count 为 1。

- 实现 `unpin_page`
  - 页不存在时返回 `false`。
  - pin count 已为 0 时返回 `false`。
  - 根据参数更新脏页标记。
  - pin count 减到 0 时，将 frame 加入 replacer。

- 实现 `flush_page`
  - 无论是否为脏页，都强制写回磁盘。
  - 写回后清除脏页标记。

- 实现 `new_page`
  - 找可用 frame。
  - 调用 `disk_manager_->allocate_page` 分配新页号。
  - 更新 frame 元信息并返回新页。

- 实现 `delete_page`
  - 页不在缓冲池中时直接返回 `true`。
  - 若页仍被 pin，则返回 `false`。
  - 脏页删除前写回磁盘。
  - 删除页表映射，重置 frame，并放回 `free_list_`。

- 实现 `flush_all_pages`
  - 遍历缓冲池中指定 fd 的所有有效页。
  - 全部写回磁盘并清除脏页标记。

### 4. `src/record/rm_file_handle.cpp`

实现记录文件操作。

修改点：

- 实现 `get_record`
  - 获取指定页。
  - 检查 slot 是否有效且 bitmap 已置位。
  - 拷贝记录内容生成 `RmRecord` 返回。
  - 操作完成后 unpin 页面。

- 实现 `insert_record(char *buf, Context *context)`
  - 调用 `create_page_handle` 获取一个有空闲 slot 的页面。
  - 通过 bitmap 找第一个空闲 slot。
  - 写入记录数据，更新 bitmap 和页内记录数。
  - 若页面插入后已满，从空闲页链表中移除。
  - unpin 页面并标记为脏页。

- 实现 `insert_record(const Rid &rid, char *buf)`
  - 用于恢复或回滚场景下向指定位置插入记录。
  - 必要时创建新页直到目标页存在。
  - 更新 bitmap、页头和空闲页链表。

- 实现 `delete_record`
  - 获取记录所在页面。
  - 检查记录是否存在。
  - 清除 bitmap，减少页内记录数。
  - 若页面从满页变为非满页，调用 `release_page_handle` 加回空闲页链表。

- 实现 `update_record`
  - 获取记录所在页面。
  - 检查记录存在后覆盖 slot 内容。
  - unpin 并标记页面为脏页。

- 实现 `fetch_page_handle`
  - 检查页号是否合法。
  - 通过缓冲池获取页面。
  - 封装为 `RmPageHandle` 返回。

- 实现 `create_new_page_handle`
  - 通过缓冲池创建新页。
  - 初始化页头、bitmap。
  - 更新文件头的 `num_pages` 和 `first_free_page_no`。

- 实现 `create_page_handle`
  - 若没有空闲页则创建新页。
  - 否则获取 `first_free_page_no` 指向的页面。

- 实现 `release_page_handle`
  - 将从满页变为非满页的页面插入空闲页链表头。

### 5. `src/record/rm_file_handle.h`

修改 `is_record`。

修改点：

- 原函数通过 `fetch_page_handle` 获取页面后没有 unpin。
- 增加查询 bitmap 后调用 `buffer_pool_manager_->unpin_page(..., false)`。
- 避免随机检查记录时不断增加 pin count，导致缓冲池页面无法被淘汰。

### 6. `src/record/rm_scan.cpp`

实现记录扫描器。

修改点：

- 构造函数初始化 `rid_` 为第一条记录页前的位置，然后调用 `next` 定位第一条有效记录。
- `next`
  - 从当前 rid 的下一个 slot 开始查找 bitmap 中为 1 的位置。
  - 当前页找不到则进入下一页。
  - 扫描到文件末尾时设置 `rid_.page_no = file_hdr_.num_pages`。
- `is_end`
  - 判断当前页号是否已经大于等于文件页数。

### 7. `src/common/config.h`

修改点：

- 增加 `#include <string>`。
- 原文件使用了 `std::string` 定义常量，但没有直接包含 `<string>`，完整编译时可能报错。

## 四、关键实现细节

### 1. 页与 frame 的关系

缓冲池使用 `page_table_` 维护 `PageId -> frame_id` 的映射。每次页面被读取或新建时，都要同步更新页表和页面元数据。

### 2. pin count 与替换策略

只有 `pin_count_ == 0` 的页面才能进入 LRU 替换器。页面被 `fetch_page` 命中时，需要从 LRU 中移除，防止正在使用的页面被淘汰。

### 3. 脏页处理

页面被修改时通过 `unpin_page(..., true)` 标记脏页。发生替换、删除、关闭文件或显式 flush 时，需要将脏页写回磁盘。

### 4. 记录页空闲链表

记录文件通过 `file_hdr_.first_free_page_no` 指向第一个有空闲 slot 的页面。页面满时从链表移除；满页删除记录后重新加入链表。
