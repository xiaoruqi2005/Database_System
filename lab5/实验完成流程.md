# Lab 5 实验完成流程

## 1. 实验目标

根据 `lab5.md`，在查询执行基础上实现唯一索引功能，支持：

- `create index table_name(col,...)`
- `drop index table_name(col,...)`
- `show index from table_name`
- 单列索引和联合索引
- 等值查询、范围查询和联合索引最左前缀匹配
- insert/delete/update 时同步维护索引并检查唯一性

## 2. 实现思路

本次采用内存有序索引实现，索引元数据仍保存在 `db.meta` 中。数据库打开时根据表数据重建内存索引，运行期间使用 `std::map<std::string, Rid>` 保存索引键到记录位置的映射。

索引键由索引列的原始二进制内容拼接而成，保证 INT、BIGINT、DATETIME、FLOAT、CHAR 等类型都能统一处理。查询时由 planner 按最左前缀选择索引，`IndexScanExecutor` 通过内存索引得到候选 RID，再用原条件判断逻辑过滤，保证范围条件和非索引列条件正确。

## 3. 修改内容

- `src/parser/ast.h`、`src/parser/yacc.y`：新增 `show index from table_name` 语法。
- `src/optimizer/plan.h`、`src/optimizer/planner.cpp`：新增 `T_ShowIndex`，并按索引最左列选择索引扫描。
- `src/system/sm_manager.h/.cpp`：实现索引元数据维护、内存索引重建、唯一性检查、索引扫描、`show_index` 输出。
- `src/execution/executor_index_scan.h`：实现索引扫描执行器。
- `src/execution/executor_insert.h`：插入前检查唯一性，插入后同步内存索引。
- `src/execution/executor_delete.h`：删除记录前同步删除索引项。
- `src/execution/executor_update.h`：更新前检查唯一性，更新后替换索引项。
- 同时保留当前主线已有的 BIGINT 和 DATETIME 支持。
