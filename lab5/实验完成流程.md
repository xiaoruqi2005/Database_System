## 3. 实现思路

### 3.1 B+树索引核心算法
- 原有流程：ix_index_handle.cpp 中所有核心函数均为 TODO/throw NotImplementedError
- 实现方案：
  - `lower_bound`：二分查找第一个 >= target 的 key
  - `upper_bound`：二分查找第一个 > target 的 key
  - `insert`：在有序位置插入键值对，重复 key 不插入（唯一索引）
  - `insert_pairs`：在指定位置批量插入键值对，处理内存移动
  - `erase_pair`：删除指定位置的键值对
  - `find_leaf_page`：从根节点遍历到叶子节点
  - `split`：节点分裂，右半部分移动到新节点，维护叶子链表和父指针
  - `insert_into_parent`：分裂后向上插入 separator key
  - `insert_entry`：完整插入流程（查找→插入→检查溢出→分裂→递归）
  - `delete_entry`：完整删除流程（查找→删除→检查下溢→合并/重分配）
  - `coalesce_or_redistribute`：删除后决定合并还是重分配
  - `adjust_root`：根节点调整
  - `redistribute` / `coalesce`：节点间重新分配或合并
- 关键数据结构：B+树节点（IxNodeHandle），文件头（IxFileHdr），页头（IxPageHdr）
- 边界条件：空树、根节点分裂/收缩、叶子链表维护、唯一性检查

### 3.2 SmManager 索引管理
- 原有流程：create_index/drop_index 为空函数体
- 实现方案：
  - `create_index`：验证表/列存在→创建索引文件→打开索引句柄→添加元数据→标记列索引→刷新元数据
  - `drop_index`：验证存在→关闭句柄→删除文件→清理元数据→刷新
  - `show_index`：遍历 IndexMeta 输出格式化的索引信息
- 关键数据结构：IndexMeta（索引列元数据）, ihs_（索引句柄映射）

### 3.3 Parser 语法补充
- 原有流程：ast.h 缺少 ShowIndex AST 节点，yacc.y 缺少 SHOW INDEX 语法规则
- 实现方案：
  - ast.h：添加 `ShowIndex` 节点（包含 tab_name）
  - ast.h：将 `JoinType` 枚举移到 `namespace ast` 外部（修复编译错误）
  - yacc.y：在 dbStmt 中添加 `SHOW INDEX FROM tbName` 规则
- 边界条件：大小写不敏感的 INDEX 关键字（lex.l 已有）

### 3.4 Planner/Execution Manager 集成
- plan.h：添加 T_ShowIndex 枚举值
- planner.cpp：在 do_planner 中添加 ShowIndex AST 的处理分支
- execution_manager.cpp：在 run_cmd_utility 中添加 T_ShowIndex case
